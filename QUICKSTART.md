# Quick Start

## Why This Matters

This package keeps curiosity useful. It prevents a single interesting motion effect from becoming an unsupported claim about energy, gravity, propulsion, or over-unity operation.
This is the shortest safe path for a new reviewer, builder, or agent.

## First 2 Minutes

1. Read this file.
2. Read `AGENT_BRIEF.md`.
3. Read `START_HERE.md`.
4. Do not claim the SEG works.
5. Do not run energy, cooling, weight, propulsion, or high-voltage tests first.

## First Action

Run only the first-gate contactless-rotation test:

```text
protocols/MVP_Contactless_Rotation_Protocol_v1.4.2.md
protocols/MVP_Contactless_Rotation_Operator_Checklist_v1.4.2.md
```

## First Run Folder

Copy or mirror this template:

```text
results/runs/RUN-TEMPLATE/
```

Create a real run folder named like this:

```text
results/runs/RUN-YYYYMMDD-001/
```

## Minimum Evidence Package

A run is not evidence until it includes:

- completed trial log,
- completed build sheet,
- completed conditioning sheet,
- raw video or motion data,
- before and after field maps,
- instrument or calibration notes,
- evidence manifest,
- SHA-256 hashes,
- reviewer signoff.

## Do Not Update Claims Until

The Claim Separation Matrix may be updated only after the run folder contains raw data, hashes, manifest, and reviewer signoff.

## Core Reminder

Narrative confidence is not evidence.
